Celectric Website Redesign

Phase 1A Submission

Strategy and scope proposal for rebuilding Celectric as an industrial product catalog and RFQ-first enquiry platform, using a stronger catalog structure inspired by Instrumart while keeping Celectric’s own branding and technical-sales positioning.

Project DirectionCatalog + Enquiry / RFQ
Reference BasisInstrumart structure + Celectric branding
Primary GoalBetter product discovery and lead capture
Submission FormatHTML summary for client review
1. Recommended Positioning

Celectric should not stay as a brochure-style site

The recommended direction is to rebuild Celectric as a structured industrial catalog with a strong enquiry workflow, instead of a simple corporate profile website.

  • Make product discovery much easier through categories, brands, search, and specs-led browsing.
  • Use RFQ / Request Quote as the core conversion path for Phase 1.
  • Support technical trust with specifications, documents, and service messaging.
  • Keep the UI professional, practical, and sales-assistive rather than overly decorative.
2. Core UX Direction

Use an Instrumart-style structure with Celectric identity

The best fit is Celectric branding on top of a more structured catalog/support framework. This means keeping Celectric’s industrial tone, logo colours, and RFQ-first model while modernising hierarchy, navigation, product listing flow, and CTA placement.

Keep: Celectric branding, technical-sales tone, calibration/support messaging, RFQ model.
Modernise: navigation, category pages, search prominence, product page structure, and repeated support CTAs.

4. Product Listing Recommendation

Category pages should behave like a real catalog

  • Clear category titles and breadcrumbs
  • Filters by brand, type, and key specifications
  • Product cards with image, model, brand, and short summary
  • Sort controls and result counts
  • Strong RFQ / Contact Sales support near browsing flow
5. Product Detail Recommendation

Detail pages should be technical, clear, and conversion-ready

  • Product title, model, brand, and image gallery
  • Short key specification summary near the top
  • Downloads / datasheets section
  • Related products or alternatives
  • Main CTA: Request Quote / Request Price / Talk to Sales
6. SEO Rule for Product Options

Minor options should usually stay under one main product page

Most small variations such as size, cable length, connector type, or rating should not become separate indexable pages. The default SEO pattern should be one canonical product page with option state handled inside the page.

  • Main page stays canonical and appears in sitemap
  • Option states may use on-page selectors or query parameters
  • Query parameter URLs should not be treated as standalone SEO pages
7. Search & Variant Discovery

Search should still understand variant-style keywords

Even when options are not separate pages, search and autocomplete should support phrases like model + option combinations.

  • Examples: product name, model number, variant terms, brand, category, aliases
  • Autocomplete can show results such as “Product A (10nm)”
  • Result should route to the parent product page with the matching option selected or highlighted
8. Product Documents

Documents should be requestable via secure download links

Recommended Handling

  • Support multiple documents per product
  • Allow product-level and variant-level documents
  • Let users select files from the product page
  • Deliver access using secure backend-generated links

Why This Is Better

  • Avoids attachment size limits
  • Reduces uncontrolled public file sharing
  • Supports request logging and follow-up
  • Creates a cleaner base for future permissions

Suggested Rule

  • Time-limited, non-guessable links
  • Recommended expiry: 24–72 hours
  • Restricted files can require internal approval
  • Internal-only files never appear in public flow
9. Confirmation Checklist

Items to confirm for final implementation planning

  • Celectric should be positioned as an industrial catalog + enquiry platform
  • Phase 1 should focus on RFQ / enquiry rather than full checkout
  • Brand pages should be included
  • Resources / document pages should be included
  • Minor options should stay within one main product page by default
  • Search should support variant-style keywords
  • Only distinct model-level variants should receive separate product pages
