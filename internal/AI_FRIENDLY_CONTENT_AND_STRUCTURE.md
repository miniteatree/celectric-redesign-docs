# AI Friendly Content and Website Structure

## Purpose
Define a content structure that is easier for AI systems, search engines, answer engines, and internal SEO workflows to understand, reuse, and summarize.

## Core Goal
Celectric should not only look professional to human visitors. Its content structure should also be easy for:
- search engines
- AI answer engines
- internal AI SEO drafting workflows
- future product data reuse across website, admin panel, and integrations

## Recommended AI Friendly Principles
- Use clear page purpose for every page
- Keep one primary topic per page
- Use structured headings in logical order
- Write short factual blocks before marketing copy
- Store product data in structured fields, not only long paragraphs
- Keep Q&A content close to product and category intent
- Support markdown-like clean content blocks in CMS/admin
- Avoid burying key specs only inside images or PDFs
- Use consistent naming for brand, model, category, and variant

## Recommended Page Structure

### Homepage
- Clear value proposition

- Short FAQ

## AI Friendly SEO Content Blocks
Each important page should support reusable structured blocks.

Suggested blocks:
- Summary
- Key specs
- Features
- Applications
- Industries served
- Compatible products or accessories
- Documents
- FAQs
- Related searches
- Enquiry CTA

## Product Q&A Recommendation
Each product should support short factual Q&A.

Example:

### Q&A Example
Q: What is Product A Spectral Sensor used for?
A: It is used for process sensing applications where optical spectral filtering is required.

Q: Does this product have multiple filter width options?
A: Yes. The sample structure includes 10nm, 20nm, and 30nm options.

Q: Are documents available by variant?
A: Yes. Some files can belong to the parent product, while others can belong to a selected variant.

Q: How are files delivered to the customer?
A: Selected files should be delivered through secure download links generated by backend.

## Markdown Friendly CMS Approach
For AI friendliness and easier reuse, content fields should support markdown-like structured writing.

Recommended content model:
- `short_description`
- `technical_summary`
- `feature_bullets`
- `application_notes`
- `faq_items`
- `document_notes`
- `seo_title`
- `seo_description`
- `seo_faq_schema_source`

Recommended formatting rules:
- prefer headings, bullets, and short paragraphs
- avoid one massive long paragraph
- avoid important facts existing only in PDF files
- avoid unclear abbreviations without explanation
- write factual answers in plain language

## Schema / Answer Engine Friendly Suggestions
Recommended structured content support:
- Product schema
- Breadcrumb schema
- FAQ schema where valid
- Organization schema
- Brand references where applicable

Important rule:
- schema content must match visible page content
- do not generate fake FAQ just for SEO

## AI Prompting Guidance for Future Content Generation
When using AI to draft page content, prompts should enforce:
- industrial B2B tone
- factual and technical clarity
- no invented specifications
- short answer blocks for FAQ
- markdown-friendly output
- SEO title and meta description generation
- optional Q&A generation from structured product data
